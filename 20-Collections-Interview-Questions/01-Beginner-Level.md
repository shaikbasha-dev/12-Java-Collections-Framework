# Java Collections Framework Interview Questions (Beginner Level)

## Introduction

This document contains the most commonly asked **Beginner-Level Java Collections Framework (JCF)** interview questions. These questions cover the basic concepts of collections and are frequently asked in interviews for **Freshers**, **Internships**, and **Entry-Level Java Developer** positions.

Understanding these questions builds a strong foundation before moving to intermediate and advanced Java Collections topics.

---

# 1. What is the Java Collections Framework (JCF)?

**Answer:**

The Java Collections Framework (JCF) is a set of interfaces, classes, and utility methods used to store, manipulate, and process groups of objects efficiently.

---

# 2. Why was the Java Collections Framework introduced?

**Answer:**

The Java Collections Framework was introduced to overcome the limitations of arrays and to provide reusable data structures and algorithms.

---

# 3. What is a Collection in Java?

**Answer:**

Collection is the root interface of the Java Collections Framework that represents a group of objects.

---

# 4. Which package contains the Java Collections Framework?

**Answer:**

```java
java.util
```

---

# 5. What are the main interfaces in the Java Collections Framework?

**Answer:**

- Collection
- List
- Set
- Queue
- Deque
- Map (Separate Hierarchy)

---

# 6. Which interface is the root interface of JCF?

**Answer:**

Collection.

---

# 7. Which interface is the parent of Collection?

**Answer:**

Iterable.

---

# 8. What is the difference between Collection and Collections?

**Answer:**

- Collection is an interface.
- Collections is a utility class.

---

# 9. What is List?

**Answer:**

List is an interface that stores ordered elements and allows duplicate values.

---

# 10. Name some implementations of the List interface.

**Answer:**

- ArrayList
- LinkedList
- Vector
- Stack

---

# 11. What is Set?

**Answer:**

Set is an interface that stores only unique elements.

---

# 12. Name some implementations of the Set interface.

**Answer:**

- HashSet
- LinkedHashSet
- TreeSet

---

# 13. What is Queue?

**Answer:**

Queue is an interface used to process elements in a particular order.

---

# 14. What is Deque?

**Answer:**

Deque stands for **Double Ended Queue**.

It allows insertion and deletion from both ends.

---

# 15. Which class implements Deque?

**Answer:**

ArrayDeque.

---

# 16. What is Map?

**Answer:**

Map is an interface used to store data as key-value pairs.

---

# 17. Is Map a child of Collection?

**Answer:**

No.

Map belongs to a separate hierarchy.

---

# 18. Name some implementations of the Map interface.

**Answer:**

- HashMap
- LinkedHashMap
- TreeMap
- Hashtable

---

# 19. Which collection allows duplicate elements?

**Answer:**

List.

---

# 20. Which collection stores unique elements?

**Answer:**

Set.

---

# 21. Which collection maintains insertion order?

**Answer:**

- ArrayList
- LinkedList
- LinkedHashSet
- LinkedHashMap

---

# 22. Which collection stores sorted elements?

**Answer:**

- TreeSet
- TreeMap

---

# 23. Which collection is synchronized?

**Answer:**

- Vector
- Stack
- Hashtable

---

# 24. Which collection does not allow duplicate keys?

**Answer:**

Map implementations do not allow duplicate keys.

---

# 25. Which collection uses key-value pairs?

**Answer:**

Map.

---

# 26. What is the default capacity of ArrayList?

**Answer:**

10.

---

# 27. What is the default capacity of Vector?

**Answer:**

10.

---

# 28. What is the difference between ArrayList and LinkedList?

**Answer:**

- ArrayList uses a dynamic array.
- LinkedList uses a doubly linked list.

---

# 29. What is the difference between HashSet and TreeSet?

**Answer:**

- HashSet stores unique elements without maintaining order.
- TreeSet stores unique elements in sorted order.

---

# 30. Why is the Java Collections Framework important?

**Answer:**

Because it provides reusable data structures, efficient algorithms, better performance, and simplifies Java programming.

# Key Takeaways

- Java Collections Framework provides reusable data structures.
- Collection is the root interface.
- List stores ordered and duplicate elements.
- Set stores unique elements.
- Queue processes elements.
- Map stores key-value pairs.
- Collections is a utility class.
- These beginner-level questions are frequently asked in fresher Java interviews.

These beginner-level interview questions cover the fundamental concepts of the Java Collections Framework. Mastering these basics helps build a strong foundation for understanding intermediate and advanced collection topics and prepares candidates for Java fresher interviews.
