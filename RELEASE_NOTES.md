# Versioning Scheme

This repository follows the Semantic Versioning (SemVer) standard.

```
MAJOR.MINOR.PATCH
```

Where:

- **MAJOR** – Incompatible or significant changes.
- **MINOR** – New features added without breaking existing content.
- **PATCH** – Documentation improvements, corrections, or minor updates.

Example:

```
1.0.0
│ │ │
│ │ └── Patch Release
│ └──── Minor Release
└────── Major Release
```
