# Bucket in Hashing

## Introduction

A **Bucket** is one of the most important concepts in Java Hashing. It is the location inside a **Hash Table** where objects are stored after their **hash code** is calculated.

Every hash-based collection in Java, such as **HashMap**, **HashSet**, **LinkedHashMap**, **LinkedHashSet**, and **Hashtable**, stores objects inside buckets.

Understanding buckets is essential for learning how Java Collections Framework performs fast insertion, searching, and deletion.

---

# Definition

A **Bucket** is a storage location inside a hash table where one or more objects are stored based on their hash code.

---

# Why are Buckets Needed?

Buckets help Java:

- Store objects efficiently
- Search objects quickly
- Insert data rapidly
- Delete objects faster
- Organize data inside a hash table

Without buckets, Java would need to search every object one by one.

---

# How Buckets Work

The bucket allocation process is:

1. An object is created.
2. Java calls the `hashCode()` method.
3. A hash code is generated.
4. The bucket index is calculated.
5. The object is stored in the corresponding bucket.

---

# Bucket Working Diagram

```text
Object
   │
   ▼
hashCode()
   │
   ▼
Hash Code
   │
   ▼
Bucket Index
   │
   ▼
Bucket
   │
   ▼
Store Object
```

---

# Example

Suppose a hash table has **10 buckets**.

```text
Hash Code = 127
```

Bucket Index:

```text
127 % 10 = 7
```

The object is stored in **Bucket 7**.

---

# Multiple Objects in a Bucket

Sometimes different objects produce the same bucket index.

Example:

```text
Object A

Hash Code = 27

Bucket = 27 % 10 = 7
```

```text
Object B

Hash Code = 37

Bucket = 37 % 10 = 7
```

Both objects are stored in **Bucket 7**.

This situation is called a **Collision**.

---

# Searching Using Buckets

When searching:

1. Java calculates the object's hash code.
2. Determines the bucket index.
3. Directly accesses that bucket.
4. Uses `equals()` if multiple objects exist in the bucket.

This makes searching much faster than checking every object.

---

# Advantages of Buckets

- Fast data access
- Efficient storage
- Quick searching
- Fast insertion
- Fast deletion
- Organized memory management

---

# Limitations

- Multiple objects may occupy the same bucket.
- Poor hash functions increase bucket collisions.
- Performance decreases if too many objects are stored in one bucket.

---

# Time Complexity

| Operation | Average Time | Worst Time |
|-----------|--------------|------------|
| Insert | O(1) | O(n) |

If many objects are stored in the same bucket, performance may degrade.

# Frequently Asked Interview Questions

### 1. What is a bucket?

**Answer:**

A bucket is a storage location inside a hash table where one or more objects are stored.

---

### 2. How is a bucket selected?

**Answer:**

Using the hash code generated by the `hashCode()` method.

---

### 3. Which method determines the bucket?

**Answer:**

```java
hashCode()
```

---

### 4. What happens if two objects are stored in the same bucket?

**Answer:**

A collision occurs, and Java uses `equals()` to identify the correct object.

---

### 5. Which Java collections use buckets?

**Answer:**

- HashMap
- HashSet
- LinkedHashMap
- LinkedHashSet
- Hashtable

---

### 6. Why are buckets important?

**Answer:**

They provide fast insertion, searching, and deletion by organizing data efficiently.

---

### 7. Can a bucket store multiple objects?

**Answer:**

Yes.

When collisions occur, multiple objects can be stored in the same bucket.

# Key Takeaways

- A bucket is a storage location inside a hash table.
- Buckets are selected using the `hashCode()` method.
- Multiple objects may exist in one bucket because of collisions.
- `equals()` identifies the correct object inside a bucket.
- Buckets enable O(1) average insertion, searching, and deletion.
- Buckets are fundamental to Java's hash-based collections.
- Frequently asked in Java Collections Framework interviews.

# Summary

A bucket is the basic storage unit of a hash table in Java. Objects are assigned to buckets using their hash codes, allowing Java to perform insertion, searching, and deletion efficiently. Although multiple objects may share the same bucket due to collisions, Java manages them effectively using `equals()`. Understanding buckets is essential for mastering the internal working of HashMap, HashSet, LinkedHashMap, Hashtable, and other hash-based collections.
