# Hash Table Concept in Java

## Introduction

A **Hash Table** is a data structure used to store data in **key-value** or **object-based** form using the concept of **hashing**. It provides very fast insertion, searching, and deletion operations by storing elements in **buckets** determined by their **hash codes**.

The Java Collections Framework uses hash tables internally in classes such as **HashMap**, **HashSet**, **LinkedHashMap**, **LinkedHashSet**, and **Hashtable**.

Understanding the Hash Table concept is essential for learning how hash-based collections work internally.

---

# Definition

A **Hash Table** is a data structure that stores elements in buckets using the hash code generated by the `hashCode()` method.

Each bucket can store one or more elements.

---

# Why is a Hash Table Needed?

Without a hash table, searching for an object requires checking every element one by one.

Example:

```text
10
20
30
40
50
```

Searching for **50** may require checking all previous elements.

A hash table directly calculates the bucket location, making searching much faster.

---

# Components of a Hash Table

A hash table consists of:

- Object
- hashCode()
- Hash Function
- Bucket
- Collision
- Rehashing

---

# Working of a Hash Table

The working process is as follows:

1. An object is created.
2. Java calls the `hashCode()` method.
3. A hash code is generated.
4. The hash function calculates the bucket index.
5. The object is stored in that bucket.
6. During searching, Java calculates the same bucket index and retrieves the object efficiently.

---

# Hash Table Working Diagram

```text
Object
   │
   ▼
hashCode()
   │
   ▼
Hash Function
   │
   ▼
Bucket Index
   │
   ▼
Store Object
```

---

# Example

Suppose the bucket size is **10**.

```text
Hash Code = 125
```

Bucket Index:

```text
125 % 10 = 5
```

The object is stored in **Bucket 5**.

---

# Searching in a Hash Table

When searching:

1. Java calculates the object's hash code.
2. The bucket index is determined.
3. The corresponding bucket is checked.
4. If multiple objects exist in the bucket, `equals()` is used to identify the correct object.

---

# Advantages

- Fast insertion
- Fast searching
- Fast deletion
- Efficient data retrieval
- Suitable for large datasets
- Widely used in Java collections

---

# Limitations

- Collisions may occur.
- Extra memory is required for buckets.
- Poor hash functions reduce performance.
- Worst-case searching becomes O(n).

# Frequently Asked Interview Questions

### 1. What is a Hash Table?

**Answer:**

A Hash Table is a data structure that stores elements in buckets using hash codes.

---

### 2. Why is a Hash Table used?

**Answer:**

To perform insertion, searching, and deletion efficiently.

---

### 3. Which Java collections use a Hash Table?

**Answer:**

- HashMap
- HashSet
- LinkedHashMap
- LinkedHashSet
- Hashtable

---

### 4. Which method generates the hash code?

**Answer:**

```java
hashCode()
```

---

### 5. Which method identifies the correct object inside a bucket?

**Answer:**

```java
equals()
```

---

### 6. What is a bucket?

**Answer:**

A bucket is a storage location inside a hash table where one or more objects are stored.

---

### 7. What happens when multiple objects are stored in the same bucket?

**Answer:**

A collision occurs, and Java uses `equals()` to locate the correct object.

---

# Key Takeaways

- A Hash Table stores data using buckets.
- `hashCode()` determines the bucket location.
- `equals()` identifies the correct object inside a bucket.
- Average insertion, searching, and deletion take O(1) time.
- Hash tables are the foundation of HashMap and HashSet.
- Collisions may reduce performance if not handled properly.
- Frequently asked in Java Collections Framework interviews.

# Summary

A Hash Table is the core data structure behind Java's hash-based collections. It uses hash codes to store objects in buckets, providing very fast insertion, searching, and deletion. By combining `hashCode()` and `equals()`, Java efficiently manages data while maintaining high performance.
