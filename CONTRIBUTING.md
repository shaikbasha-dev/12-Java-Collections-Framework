# Why Contribute?

By contributing to this repository, you help:

- Improve Java learning resources.
- Make explanations easier to understand.
- Fix errors or outdated information.
- Add useful Java programs.
- Improve interview preparation content.
- Help thousands of learners around the world.

---

# Ways to Contribute

You can contribute in many ways, including:

- Fix spelling or grammar mistakes.
- Improve explanations.
- Add additional examples.
- Improve Java source code.
- Add interview questions.
- Improve formatting.
- Add diagrams or illustrations.
- Report bugs or incorrect information.
- Suggest new topics.
- Improve documentation.

---

# Before You Start

Before making any contribution, please:

- Read the README carefully.
- Check whether a similar issue already exists.
- Follow the existing folder structure.
- Maintain consistency throughout the repository.

---

# Repository Guidelines

Please follow these guidelines while contributing:

- Keep explanations simple and beginner-friendly.
- Write clean and readable Markdown.
- Maintain the existing folder structure.
- Do not rename existing folders or files without discussion.
- Keep formatting consistent.
- Verify all Java code before submitting.
- Ensure examples are easy to understand.

---

# Coding Standards

When contributing Java programs:

- Use meaningful class names.
- Write clean and properly indented code.
- Add comments wherever necessary.
- Keep programs simple and educational.
- Include expected output whenever applicable.
- Avoid unnecessary complexity.
- Ensure programs compile successfully.

---

# Documentation Standards

Documentation should:

- Use proper Markdown formatting.
- Include clear headings.
- Explain concepts in simple language.
- Include comparison tables where appropriate.
- Add interview tips whenever useful.
- Include summaries for important topics.

---

# Commit Message Convention

Use meaningful commit messages.

Examples:

```text
docs: improve HashMap documentation

docs: add interview questions for TreeMap

feat: add ArrayList insertion program

feat: add HashSet example program

fix: correct LinkedList explanation

style: improve markdown formatting
```

---

# Pull Request Process

Before submitting a Pull Request:

1. Fork the repository.
2. Create a new branch.
3. Make your changes.
4. Test all Java programs.
5. Review your changes.
6. Commit using a meaningful message.
7. Push the branch.
8. Create a Pull Request.

Please include:

- Description of changes
- Reason for changes
- Screenshots (if documentation changes significantly)

---

# Code Review Process

Every contribution may be reviewed to ensure:

- Accuracy
- Readability
- Code quality
- Documentation quality
- Consistent formatting
- Educational value

Feedback may be provided before merging.

---

# Contribution Checklist

Before submitting your contribution, make sure:

- Java code compiles successfully.
- Markdown formatting is correct.
- No unnecessary files are added.
- Existing files are not accidentally modified.
- Commit messages follow the repository convention.
- Content is original or properly referenced.
- Examples are beginner-friendly.

---

# Need Help?

If you have questions or suggestions:

- Open an Issue.
- Start a Discussion.
- Submit a Pull Request with your proposed improvements.

Every contribution, no matter how small, helps make this repository more valuable for the Java community.

Thank you for contributing and supporting open-source learning!
